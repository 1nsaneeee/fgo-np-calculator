## 二、完整计算架构

### 2.1 数据流

```
从者基础信息表 ─┐
从者ATK及hit分布表 ─┤
                  ├──> 计算器(计算器(中文)/Calc Main)
用户输入(礼装/助战/技能Buff) ─┘
                          │
                          ├──> ATK计算
                          ├──> Buff汇总(含cap)
                          ├──> 职阶克制系数查找
                          ├──> 阵营克制判定
                          ├──> 宝具伤害计算(含三种随机数)
                          ├──> 指令卡伤害计算(每T按配卡循环)
                          ├──> NP获取计算(每hit)
                          ├──> 暴击星计算(出星+分配)
                          ├──> 连发判定(鞭尸/过量击杀)
                          └──> 期望值汇总
```

### 2.2 核心单元格映射 (Calc Main / 计算器(中文))

#### 输入区 (Rows 1-20)

| 单元格 | 公式 | 含义 |
|--------|------|------|
| C1 | `="Servant HP="&VLOOKUP($A$3,'从者ATK及hit分布表'!$E:$IT,B3+130,0)` | 显示当前从者HP |
| C3 | `=VLOOKUP($A$3,'从者ATK及hit分布表'!$E:$IT,B3+14,0)` | 从者ATK(根据等级查找) |
| D3 | `=VLOOKUP($A$3,'从者基础信息表'!$B:$AD,21+RIGHT($B$5,1),0)` | 宝具倍率(根据宝具等级) |
| E3 | `=VLOOKUP($A$3,'从者基础信息表'!$B:$U,4,0)` | 宝具颜色(Quick/Buster/Arts) |
| F3 | `=VLOOKUP($A$3,'从者基础信息表'!$B:$U,5,0)` | NP获取率 |
| G3-H3~O3 | VLOOKUP系列 | 宝具Hits及各色卡NP率/Hits |

#### ATK汇总 (Row 19)

```
D19 = C3 + C5 + D5 + C7
     = 从者基础ATK + 芙芙ATK + 礼装ATK + 自定义额外ATK
```

### 2.3 等级-LV-ATK映射系统

从者ATK使用动态查找:
```
C3 = VLOOKUP($A$3,'从者ATK及hit分布表'!$E:$IT, B3+14, 0)
```

- `B3+14`: 根据输入等级的列偏移
  - LV1 ATK在列14+1=15, LV90在列14+90=104
- HP使用偏移: `B3+130` (HP列在ATK列之后)
- 支持LV1-LV120全等级精确查找

## 三、Buff/Debuff系统 - 完整公式解析

### 3.1 所有Buff类别与Cap值 (Row 19 汇总)

Buff分为**礼装(CE) / 自身(Self) / 助战(Support) / 敌人(Enemy) / 减益(Debuff)**五个来源，最终汇总为各项净Buff值。

| 列 | Buff名称 | 来源组合公式 | Cap |
|----|----------|-------------|-----|
| D | **防御力Down(敌人)** | `D5+D7+D11+D15+额外` | N/A(攻击力侧) |
| E | **攻击力UP** | `MIN((E5+E7+E11+E15-E17+E13*H137), 400)/100 + IF(D方>0, MIN(D方/100, 1), -D17/100)` | 400% |
| F | **Buster魔放** | `MIN((F5+F7+B11+F11+F15-F17+F13*H137+F9)/100, 4)*100` | 400% |
| G | **Arts魔放** | `MIN((G5+G7+G11+B13+G15-G17+G13*H137+G9)/100, 4)*100` | 400% |
| H | **Quick魔放** | `MIN((H5+H7+H11+C13+H15-H17+H13*H137+H9)/100, 4)*100` | 400% |
| I | **暴击威力UP** | `MIN(I5+I7+I11+C11+I15-I17+I13*H137, 500)` | 500% |
| J | **出星率UP** | `MIN(J5+J7+J11+J15-J17+J13*H137, 300)` | 300% |
| K | **宝具威力UP** | `IF(K方>0, MIN(K方/100, 5), -K17/100)*100` | 500% |
| L | **NP率UP** | `MIN(L方/100, 10) - MIN(L17/100, 10)` | 1000% |
| M | **C类特攻** | `MIN((M5+M7+M11+C15+M15-M17+M13*H137)/100, 4)*100` | 400% |
| N | **固定伤害** | `MIN(100000, N5+N7+N11+N15+C17+N13*H137) - MIN(100000, N17)` | 100000 |

### 3.2 攻击力Buff的详细公式

```
E19 = ((1 + MIN((E5+E7+E11+E15-E17+E13*H137), 400)/100
       + IF((D7+D11+D15+D13*H137)>0,
            MIN((D7+D11+D15-D17+D13*H137)/100, 1),
            -D17/100)
      ) - 1) * 100
```
- **攻击力UP** (E项): Cap 400%
- **防御力DOWN** (D项): 等效攻击力加成，Cap 100%

### 3.3 被动技能和开关控制 (Row 137)

| 单元格 | 开关含义 | 值 |
|--------|---------|-----|
| H137 | 追加被动技能启用 | `=IF(B9="Yes", 1, 0)` |
| I137 | 某种被动暴击/宝威启用 | `=IF(A15="Yes", 1, 0)` |
| J137 | 宝具颜色切换(被动) | `=IF(D9="Yes", 1, 0)` |
| L137 | 首卡加成考虑 | `=IF(A53="Yes", 1, 0)` |
| M137 | Overkill(鞭尸)加成 | `=IF(B53="Yes", 1, 0)` |
| N137 | 暴击启用 | `=IF(A37="Yes", 1, 0)` |

## 四、宝具伤害计算公式

### 4.1 基础伤害公式（三种随机数）

```
Row 26 最小伤害 (0.9 随机):
C26 = ROUNDDOWN(0.9 * 0.23 * (1 - O15/100 - O11/100) * D19 * (D3 + C9)/100
       * A19 * B19 * C19 * B7
       * (1 + IF(M140卡色, F19/G19/H19)/100)
       * (1 + E19/100)
       * (1 + (K19 + L19)/100)
       * H133 * (1 + (E9 - 100) * J137/100) + N19(固伤), 0)

Row 27 平均伤害 (1.0 随机):
C27 = ROUNDDOWN(0.23 * (1 - O15/100 - O11/100) * D19 * (D3 + C9)/100
       * A19 * B19 * C19 * B7
       * (1 + 对应色卡魔放/100)
       * (1 + E19/100)
       * (1 + (K19 + L19)/100)
       * (1 + (E9 - 100) * J137/100) + N19, 0)

Row 28 最大伤害 (1.099 随机):
C28 = ROUNDDOWN(1.099 * ...(同上)... + N19, 0)
```

### 4.2 伤害公式中各项含义

| 因子 | 值/来源 | 说明 |
|------|---------|------|
| **0.23** | 固定值 | FGO伤害基础系数 |
| **0.9/1.0/1.099** | 固定 | 随机数范围(±10%，正态分布近似) |
| **D19** | =总ATK | 从者ATK + 芙芙ATK + 礼装ATK + 额外ATK |
| **D3** | 宝具倍率(%) | 如1000%=1000 |
| **C9** | 宝具伤害附加 | (被动/宝威额外) |
| **(1-O15/100-O11/100)** | 防御力减伤 | 敌人防御 - 防御Down |
| **A19** | 宝具色卡倍率 | Quick=0.8, Buster=1.5, Arts=1.0 |
| **B19** | 阵营克制 | Sky>Earth=1.1, 反之=0.9, Star↔Beast=1.1 |
| **C19** | 职阶克制 | VLOOKUP查找职阶克制表(2.0/1.5/1.0/0.5等) |
| **B7** | 职阶补正系数 | 各职阶固有伤害补正 |
| **H133** | 特殊修正 | 如Super Aoko=0.5 (B126) |
| **(1+色卡魔放/100)** | 对应颜色魔放 | Buster=F19, Arts=G19, Quick=H19 |
| **(1+E19/100)** | 攻击力Buff | 含AtkUp + DefDown等效 |
| **(1+(K19+L19)/100)** | 宝威+NP率Buff | 宝具威力 + NP率对宝具伤害的加成 |
| **(1+(E9-100)*J137/100)** | 特攻倍率 | 如200=2倍, 仅J137=1时生效 |
| **N19** | 固定伤害加算 | 不受其他Buff影响的最终加算 |

### 4.3 敌人防御系统

```
敌方防御减伤 = (1 - O15/100 - O11/100)
O15 = 敌方防御力%
O11 = 敌方追加防御力%
```

## 五、指令卡伤害计算

### 5.1 指令卡位置乘数 (First Card Bonus)

```
G34 = IF(AE34="Extra", 1,
        IF(AE34="Buster", 0,
          IF(AE34="Arts", (3 + (AF34-1) * 1.5),
            IF(AE34="Quick", (1 + (AF34-1) * 0.5), ))))
```

| 位置 | Buster | Arts | Quick |
|------|--------|------|-------|
| 第1位 | 0 | 3.0 | 1.0 |
| 第2位 | 0 | 4.5 | 1.5 |
| 第3位 | 0 | 6.0 | 2.0 |

- Extra Attack固定为1.0倍

### 5.2 指令卡伤害公式 (以第1T首卡为例)

```
B40 = ROUNDDOWN(0.9 * 0.23 * (1-(O15+I32)/100) * H133 * (D19+K21)
      * (Z50 * Y50 * (1 + IF(卡色, 魔放/100)) + 0.5 * AA50)
      * C19 * B19 * B7 * H133
      * (1 + E27/100)
      * (1 + N137)
      * (1 + F32/100 + MIN(5, 色卡暴击被动/100 + I27/100) * N137)
      * B126 + H32, 0)
```

其中：
- `Y50` = 指令卡位置倍率 (1 + (position-1) * 0.2)
- `Z50` = 指令卡色卡倍率 (Buster=1.5, Quick=0.8, Arts=1.0)
- `AA50` = 暴击标记 (是/否)
- `(1+N137)` = 暴击伤害倍率 (暴击时=2.0)
- 暴击威力Cap = 500%

### 5.3 指令卡配卡系统

```
S50 = COUNTIF($K$29:$M$29, "Buster")   -- 宝具数Buster
S51 = COUNTIF($K$29:$M$29, "Arts")     -- 宝具数Arts
S52 = COUNTIF($K$29:$M$29, "Quick")    -- 宝具数Quick

T52 = IF(S52=3, "QQQ",
        IF(S52=2, VLOOKUP(S50, Q45:S46, S51+2, FALSE),
          IF(S52=1, VLOOKUP(S50, Q38:T40, S51+2, FALSE),
            VLOOKUP(S50, Q31:U34, S51+2, FALSE))))
```

配卡序列存储在Q31:U34查找表中，支持所有15种配卡组合。

## 六、NP获取计算公式

### 6.1 单卡NP获取 (以Row 55为例)

指令卡NP获取:
```
E55 = ROUNDDOWN(IF(卡色="Extra",
                hits * (NPperHit * 1 + 1 * critMod) * enemyMod * (1 + M19/100),
                hits * (NPperHit * (1 + 对应魔放/100) + 1 * critMod)
                     * enemyMod * (1 + M19/100) * (1 + 1 * L137)), 2)
      * (1 + 0.5 * M137)

F55 = E55 * IF(卡色="Buster", I3, IF(卡色="Arts", K3, IF(卡色="Quick", M3, IF(卡色="Extra", O3, ))))
```

NP获取 = 单HitNP量 × 总Hits数

各参数：
- `hits(X3/L3/M3)` = 该色卡总Hit数
- `NPperHit(AD34/X35等)` = 每Hit的NP率
- `AH34/AB35等` = 敌方NP补正 = `HLOOKUP(敌职阶, B144:Z145, 2, 0) * (1 + 0.2 * I137)`
- `M19` = C类特攻(NP获取侧)
- `M137` = 暴击时NP加成(0.5倍)
- `L137` = 首卡NP加成

### 6.2 色卡NP率Buff (Row 20)

```
F20 = MIN((CE-Buster + Support-Buster + 敌Buster + Own-Buster - Debuff + Passive*H137)/100, 4) * 100
G20 = MIN((CE-Arts + Support-Arts + 敌Arts + Own-Arts - Debuff + Passive*H137)/100, 4) * 100
H20 = MIN((CE-Quick + Support-Quick + 敌Quick + Own-Quick - Debuff + Passive*H137)/100, 4) * 100
```

各色卡NP率的魔放加成，Cap 400%。

### 6.3 Extra Attack NP

Extra卡NP计算特殊处理:
```
AD49 = IF(AB49="Extra", 1, 1+(AC49-1)*0.2)  -- Extra NP位置系数恒为1
AE49 = IF(AB49="Buster", 1.5, IF(AB49="Quick", 0.8, 1))  -- Extra色卡系数继承首卡
```

### 6.4 宝具NP获取 (Row 25)

```
B25 = IF(J29=0, B24,                        -- 无NP需求: 全Hit
        IF(J29 - C27 > 0, B24,              -- NP不足: 全Hit
          MIN(B24, MATCH(1, N107:N121, 0)-1))) -- 刚好够NP: 精确Hit数

C25 = MIN(B25, G3) * ROUNDDOWN(每HitNP_Overkill修正)
      + MAX(0, (G3 - B25)) * ROUNDDOWN(每HitNP_Overkill修正 * 1.5)
```

**鞭尸(Overkill)系统**:
- 前半Hit: 按普通NP率计算
- 后半Hit (从刚好打死开始): NP率 x 1.5倍
- `K133 = VLOOKUP(M140, A155:B157, 2, 0)` = 宝具NP系数 (Arts=3, Quick=1, Buster=0)

## 七、暴击星计算

### 7.1 基础出星率 (Row 29-31)

```
C29 = ROUNDDOWN(MIN(300, B15*100 + 色卡基础出星 + 出星Buff + O19)/100, 0) * MIN(B25, G3)
      + MAX(0, (G3-B25)) * ROUNDDOWN(MIN(300, ... + 30(Overkill加成))/100, 0)

C30 = MIN(300, ...)/100 * MIN(B25, G3)
      + MAX(0, (G3-B25)) * MIN(300, ... + 30)/100

C31 = MIN(3, ROUNDUP(MIN(300, ...)/100, 0)) * MIN(B25, G3)
      + MAX(0, (G3-B25)) * MIN(3, ROUNDUP(MIN(300, ... + 30)/100, 0))
```

三栏分别表示:
- C29: 保底出星 (ROUNDDOWN)
- C30: 期望出星 (精确值)
- C31: 最大出星 (ROUNDUP, Cap=3/Hit)

### 7.2 色卡基础出星值 (Row 65-70)

```
AD65 = IF(AB65="Quick", 80 + 50*(AC65-1),     -- Quick: 80+50/130/180/230
         IF(AB65="Buster", 10 + 5*(AC65-1),   -- Buster: 10+5/15/20/25
           IF(AB65="Extra", 100, 0)))           -- Extra: 100, Arts: 0
```

### 7.3 出星率汇总公式

```
AF65 = MIN(300, B15*100 + AD65*(1+对应魔放/100) + J19 + 20 + O19 + 20*W210 + 30*X210)
```

- `B15*100` = 从者基础出星率(%)
- `AD65*(1+魔放/100)` = 色卡Hits出星值 × 魔放
- `J19` = 出星率Buff (Cap 300)
- `O19` = 敌人出星率补正
- `W210/X210` = 首卡/暴击出星加成

### 7.4 暴击星分配结果

```
F70 = ROUNDDOWN(AF65/100, 0) * Hits   -- 保底暴击星
G70 = AF65/100 * Hits                 -- 期望暴击星
H70 = MIN(3, ROUNDUP(AF65/100, 0)) * Hits  -- 最大暴击星(每Hit上限3星)
```

## 八、阵营克制与职阶克制

### 8.1 阵营克制 (Row 19)

```
B19 = IF(AND(天-Sky, 地-Earth), 1.1,
        IF(AND(天-Sky, 人-Human), 0.9,
          IF(AND(地-Earth, 人-Human), 1.1,
            IF(AND(地-Earth, 天-Sky), 0.9,
              IF(AND(人-Human, 天-Sky), 1.1,
                IF(AND(人-Human, 地-Earth), 0.9,
                  IF(AND(星-Star, 兽-Beast), 1.1,
                    IF(AND(兽-Beast, 星-Star), 1.1,
                      1))))))))
```

克制关系:
- 天>地>人>天 (1.1倍)，逆向0.9倍
- 星↔兽 (1.1倍互相)
- 其余=1.0

### 8.2 职阶克制 (Row 19 C列)

```
C19 = VLOOKUP(A7, $B$281:$AA$305, 1 + MATCH(A11, $C$280:$AA$280, 0), FALSE) + C21
```

职阶克制系数表存储在 `B281:AA305`:
- 常规：Saber>Lancer>Archer>Saber (2.0x)
- Berserker: 对所有1.5x, 被所有1.5x
- Ruler: 对常规6职阶1.1x减伤
- Avenger: 对Ruler 2.0x
- Alter Ego: 对 Cavalry三骑 1.5x
- Foreigner: 对Berserker 2.0x, 互相0.5x
- Pretender: 对Alter Ego 2.0x, 对3骑士0.5x
- Beast: 特殊克制表

### 8.3 职阶补正系数

```
B7 = HLOOKUP(A7, $B$144:$Z$146, 3, 0)
```

各职阶固有伤害系数: Saber=1.0, Archer=1.0, Lancer=1.05, Caster=0.9, Assassin=0.9, Berserker=1.1, Ruler=1.1, Avenger=1.1, etc.

## 九、连发(宝具回NP)系统

### 9.1 宝石/魔放判定逻辑

在每回合计算中，通过MATCH函数判断NP回收是否足够再开宝具:

```
K39 = NP需求量(通常=100)
K48 = NP回收量(宝具+前指令卡NP)

K85 = IF(K39=0, B24,                         -- 无需NP: 满Hit
         IF(K39 - K48 > 0, B24,               -- NP不够: 满Hit
           MIN(B24, MATCH(1, G91:G105, 0)-1))) -- NP刚好: 精确Hit截断
```

NP累积表(Row 91-121):
```
E91-E121: 累计宝具伤害百分比(从宝具Hit分布解析)
F91-F121: ROUND(K48/100 * E91, 0) = 该Hit后累计NP回收
G91-G121: IF(F91 - K39 >= 0, 1, 0) = 是否达到NP线
```

### 9.2 鞭尸Hit判定

```
N107-N121: 并行计算另一侧NP回收表(用于判定第2/3面)
M107-M121: ROUND(C27/100 * L107, 0) = 非暴击NP累积
N107-N121: IF(M107 - J29 >= 0, 1, 0) = 达到NP线标记
```

### 9.3 宝具NP回收公式 (Row 133)

```
N133 = ROUNDDOWN(L140 * K133 * (1 + 宝具对应魔放/100) * L133
                 * (1 + M19/100) * (1 + 0.5 * K131), 2)
O133 = N133 * B24
```

- `L140` = NP率(F3)
- `K133` = 宝具NP系数(Arts=3, Quick=1, Buster=0, 查表A155:B157)
- `L133` = 敌方NP补正
- `K131` = overkill标记

## 十、ATK/HP推算系统

### 10.1 物理型从者

```
最大ATK = 星级基础值 × 职阶成长倾向 × 攻防倾向 × 筋力补正 × 敏捷魔力平均补正
         × [1 - (筋力补正-1) × (敏捷补正-1)]

敏捷魔力平均 = (敏捷补正 + 魔力补正) / 2
```

### 10.2 魔法型从者

```
最大ATK = 星级基础值 × 职阶成长倾向 × 攻防倾向 × 魔力补正
```

### 10.3 星级基础值

| 星级 | ATK基础 | HP基础 |

| 1星 | 5500 | 7500 |
| 2星 | 6200 | 8500 |
| 3星 | 7000 | 10000 |
| 4星 | 9000 | 12500 |
| 5星 | 11000 | 15000 |

### 10.4 攻防倾向

| 类型 | 倍率 |
|------|------|

| 均衡 | 1.0 |

## 十一、期望值计算系统

### 11.1 多T循环期望 (Row 170+)

计算器支持最多3T(3回合)的期望伤害/NP计算:

**伤害期望**:
```
E172 = ROUNDDOWN(IF(卡Extra, hits*(NPperHit*1*...)*enemyMod*(1+M19/100),
                    hits*(NPperHit*(1+对应魔放/100)+...)*enemyMod*(1+M19/100)*(1+L137)), 2)
       * (1 + 0.5 * B170)   -- B170=是否暴击
```

在每个Turn计算中遍历:
- A170/H170/O170/V170: 各列对应不同分支场景
- 5列并行计算: 无暴击/暴击/首卡暴击/助战暴击/...

### 11.2 概率权重

每种配卡模式都有对应的概率，通过多分支加权求和得到最终期望值。

## 附录: 公式速查卡

### 宝具期望伤害
```
Damage = 0.23 × ATK × NP倍率% × 色卡倍率
         × 阵营系数 × 职阶克制 × 职阶补正
         × (1 + 色卡魔放%) × (1 + AtkUp%)
         × (1 + 宝具威力% + NP率Buff%)
         × 特攻倍率 × 随机数
         - 防御减伤 + 固伤
```

### 指令卡伤害
```
CardDamage = 0.23 × ATK × 位置倍率 × 色卡倍率 × 首卡加成
             × 阵营 × 职阶 × 补正
             × (1 + 魔放%) × (1 + AtkUp%)
             × (1 + 暴击威力%(if crit)) × (1 + 宝威%)
             × 随机数 + 固伤
```

### NP获取(每Hit)
```
NP_per_Hit = NP率 × 色卡NP系数 × (1 + 对应魔放%)
             × 敌方补正 × (1 + NP率Buff%)
             × 首卡加成 × Overkill加成
```

### 暴击星(每Hit)
```
Star_per_Hit = MIN(300, 基础出星率 + 色卡出星值 × (1 + 魔放%)
              + 出星率Buff + 敌补正) / 100
Stars = Σ Star_per_Hit × Hits
```
